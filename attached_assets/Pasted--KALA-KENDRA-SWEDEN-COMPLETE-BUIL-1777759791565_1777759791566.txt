# =============================================================
# CONTEXT — THE NATARAJA HERITAGE THEME
# =============================================================

This site honours classical Bharatanatyam, Carnatic music, and Kerala temple aesthetics. The design must feel rooted in tradition, dignified, and warm — never generic or tech-trendy.

## Color Palette (Tailwind theme.extend.colors)

```
maroon: { DEFAULT: '#5C1416', deep: '#3D0A0C' }
gold: { DEFAULT: '#B8893A', deep: '#8B6520' }
stone: { DEFAULT: '#8B7355', light: '#D4C4A0' }
parchment: '#F4EBD9'
ink: '#1F1612'
soft: '#5A4A42'
saffron: '#C25E1F'
moss: '#5F7A3F'
rust: '#8C2F1A'
```

NEVER use pure white (#FFFFFF) or pure black (#000000). Use parchment for white and ink for black.

## Typography (next/font/google)

```
display: Cormorant Garamond — for ALL headings (h1, h2, h3, h4)
sans: Manrope — for body text, buttons, navigation
hand: Caveat (cursive) — for occasional decorative italic accents
```

Heading weights: h1/h2 = 400, h3 = 500. Body line-height: 1.6.

## Decorative Motifs

- Three-star ornament dividers: ❋   ❋   ❋ (centered, gold, between sections)
- Drop caps on opening paragraphs (large Cormorant first letter, maroon, floated left)
- Gold border frames around photos (offset diagonally by 12-16px)
- Section eyebrows: small uppercase gold label preceded by a short gold rule, above each heading
- Pull quotes: italic Cormorant maroon with 2px gold left border
- Subtle stone texture: fixed body::before with radial gradient overlays of gold (6% opacity) and maroon (4% opacity)

## Layout

- Generous negative space (sections: 7rem padding desktop, 5rem mobile)
- Asymmetric over symmetric
- Single-column on mobile (360px minimum), two-column on desktop
- Sharp corners on buttons (border-radius: 0)
- Slow, intentional animations (0.25s for hover, 0.4s for transitions)
- Respect prefers-reduced-motion

## Buttons

```
.btn-primary { background: maroon; color: parchment; uppercase; tracking-wider; sharp corners; 1px maroon border }
.btn-ghost { transparent background; maroon text; maroon border; uppercase }
```

## Status badge colors (semantic)

```
pending → stone
under_review → saffron
accepted / active / paid → moss
waitlisted → gold-deep
rejected / inactive → rust
```

## Icons

Use lucide-react throughout. Color them gold-deep for decorative, maroon for active.

## Phase 1 Tables

### admins
```
id uuid PK, email varchar(160) unique, full_name varchar(120),
role varchar(20) [owner|teacher], active boolean default true,
last_login_at timestamp, created_at timestamp default now()
```

### admissions
```
id uuid PK
applicant_type varchar(10) [adult|child]
student_name varchar(120) NOT NULL
student_dob date NOT NULL
student_gender varchar(20)
student_email varchar(160) -- if adult
student_phone varchar(40) -- if adult
parent_name varchar(120) -- if child
parent_relationship varchar(20)
parent_email varchar(160)
parent_phone varchar(40)
emergency_name varchar(120)
emergency_phone varchar(40)
address_street varchar(200) NOT NULL
address_postal varchar(20) NOT NULL
address_city varchar(80) NOT NULL
batch varchar(40) NOT NULL
experience varchar(20) [none|some|significant]
experience_details text
joining_date date
medical_notes text
will_stage_perform varchar(10) [yes|no|maybe] NOT NULL
motivation text
referral_source varchar(40)
photo_consent varchar(20) [yes-all|yes-internal|no] NOT NULL
rules_consent varchar(10) [agree|disagree] NOT NULL
suggestions text
status varchar(20) default 'pending' [pending|under_review|accepted|waitlisted|rejected]
reviewed_by uuid FK admins.id
reviewed_at timestamp
admin_notes text
submitted_at timestamp default now()
created_at timestamp, updated_at timestamp
```

### students
```
id uuid PK, admission_id uuid FK admissions.id,
full_name varchar(120), dob date, batch_id uuid FK batches.id,
primary_contact_name varchar(120), primary_contact_email varchar(160),
primary_contact_phone varchar(40), status varchar(20) [active|inactive|withdrawn],
enrolled_at timestamp, created_at, updated_at
```

### batches
```
id uuid PK, code varchar(20) unique, name varchar(80),
age_range varchar(40), description text,
active boolean default true, display_order integer
```

### settings (single-row table, id always = 1)
```
id integer PK, monthly_fee_sek integer default 400,
school_name varchar(120), contact_email varchar(160),
contact_phone varchar(40), address_line varchar(200),
accepting_applications boolean default true, updated_at timestamp
```

### audit_log
```
id uuid PK, admin_id uuid FK admins.id, action varchar(40),
entity_type varchar(20), entity_id uuid, details jsonb,
created_at timestamp
```

## Phase 2 Tables

### schedule_slots
```
id uuid PK, batch_id uuid FK batches.id, weekday smallint (0=Mon, 6=Sun),
start_time time, end_time time, location varchar(120) default 'Masala Kitchen',
active boolean default true, effective_from date, effective_until date
```

### sessions
```
id uuid PK, batch_id uuid FK batches.id, schedule_slot_id uuid FK,
session_date date, start_time time, end_time time,
status varchar(20) [scheduled|cancelled|completed], notes text,
created_at, updated_at
```

### attendance
```
id uuid PK, session_id uuid FK sessions.id, student_id uuid FK students.id,
status varchar(15) [present|absent|excused|late], notes text,
marked_by uuid FK admins.id, marked_at timestamp
```

### email_log
```
id uuid PK, recipient_email varchar(160), template varchar(40),
subject varchar(200), status varchar(20) [sent|failed|pending],
error text, sent_at timestamp
```

## Phase 3 Tables

### fees
```
id uuid PK, student_id uuid FK students.id, period_year smallint,
period_month smallint (1-12), amount_sek integer,
status varchar(20) [due|paid|waived|partial], paid_amount_sek integer default 0,
paid_at timestamp, payment_method varchar(20) [cash|swish|transfer],
notes text, created_at, updated_at
```

### events
```
id uuid PK, name varchar(120), event_date date, venue varchar(200),
description text, status varchar(20) [planning|registration_open|closed|completed],
created_at, updated_at
```

### event_participants
```
id uuid PK, event_id uuid FK events.id, student_id uuid FK students.id,
role varchar(80), piece_name varchar(120), confirmed boolean default false,
notes text, created_at, updated_at
```
